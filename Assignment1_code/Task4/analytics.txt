
The envirotrack.db database (generated by SensorMonitor.py) stores one
row every 10 seconds, containing:

  • Timestamp                  – datetime of the reading
  • Temperature (raw + cal)    – °C, calibrated by subtracting 2°C offset
  • Humidity                   – percentage (%)
  • Pressure                   – hectopascals (hPa)
  • Pitch / Roll / Yaw         – orientation angles in degrees
  • Status labels              – classification for each sensor reading
                                 (e.g. "Low", "Comfortable", "High",
                                       "Aligned", "Tilted")

This gives two natural visualisation questions:
  Q1: How do sensor values change over time? → Time-series line chart
  Q2: How often is each classification reached? → Status count bar chart

Chart 1 – Time-Series Line Chart (matplotlib)
----------------------------------------------
  X-axis : Timestamp (HH:MM format)
  Y-axis : Temperature (°C) and Humidity (%) – left axis
           Pressure (hPa) – right axis (twin axis)
  Lines  : Three separate lines for temperature, humidity, pressure.

  Decision rationale:
  A line chart is the most natural choice for continuous time-series
  sensor data because it reveals trends, spikes, and gradual drift in
  environmental conditions over the monitoring period. Scatter plots
  would lose the continuity implied by sequential readings; bar charts
  would obscure temporal ordering. Dual y-axes are necessary because
  pressure (980–1030 hPa) operates on a completely different numerical
  scale than temperature (15–30 °C) or humidity (30–65 %).

Chart 2 – Status Distribution Bar Chart (seaborn)
--------------------------------------------------
  X-axis : Sensor type (Temperature, Humidity, Pressure, Orientation)
  Y-axis : Count of readings per status label
  Hue    : Status category (Low / Comfortable / Normal / High /
           Aligned / Tilted)
  Colours: Match the Sense HAT LED scheme (blue=Low, green=Normal,
           red=High, amber=Tilted)

  Decision rationale:
  A grouped bar chart is ideal for comparing categorical frequency
  distributions across multiple discrete groups. It immediately
  communicates how stable the environment was (mostly "Comfortable")
  versus how often thresholds were breached. A pie chart was considered
  but rejected as it cannot easily compare across four different
  sensor groups simultaneously. A histogram was unsuitable because the
  status labels are categorical (not continuous).

Library 2 – Seaborn (v0.13.x)

The combination provides the best of both worlds: Seaborn's statistical capabilities
and attractive defaults with matplotlib's fine-grained control when needed.

Question: Which two libraries did you narrow it down to?

Answer: Matplotlib and Seaborn

Matplotlib
──────────────
Overview:
  Matplotlib is the foundational Python plotting library, released in
  2003 by John D. Hunter. It provides a MATLAB-like stateful API
  (pyplot) and a full object-oriented API for complete plot control.

Advantages:
  • Maximum flexibility – every element (axes, ticks, labels, legends,
    grid lines, figure size, DPI) can be customised programmatically.
  • Supports complex multi-axis layouts (e.g. twin y-axes used here),
    subplots, insets, and 3D plots.
  • Non-interactive "Agg" backend allows server-side rendering without
    a display, ideal for Raspberry Pi headless operation.
  • Extensive documentation and community support spanning 20+ years.
  • Fine-grained control over file format and resolution (PNG, SVG,
    PDF, etc.).

Disadvantages:
  • Verbose syntax: even a simple styled chart requires 15–25 lines of
    boilerplate (figure creation, axis labels, legend, tight_layout).
  • Default aesthetics are plain and require significant styling effort
    to produce publication-quality visuals.
  • No built-in statistical abstractions (grouping, aggregation); the
    developer must manually prepare data before plotting.
  • Twin-axis management (twinx/twiny) is unintuitive for beginners.

Complexity:
  Medium-to-high. Suitable for developers comfortable with an explicit,
  imperative plotting model.

Flexibility / Configuration:
  Very high – essentially unlimited customisation. Any chart type
  imaginable can be constructed given sufficient effort.


Seaborn
───────────
Overview:
  Seaborn (v0.13) is a statistical visualisation library built on top
  of Matplotlib. It provides a declarative, high-level API that maps
  DataFrame columns directly to visual attributes (x, y, hue, size).

Advantages:
  • Concise API: a grouped bar chart with colour-coded hue requires
    only 5–8 lines compared to 20+ in pure Matplotlib.
  • Built-in statistical aggregations (mean, confidence intervals,
    counts) applied automatically based on the plot type chosen.
  • Attractive default themes (despine, palette management) that
    produce professional visuals with minimal configuration.
  • Native integration with Pandas DataFrames – columns are referenced
    by name, making code highly readable and maintainable.
  • Colour palettes are semantically meaningful and perceptually
    uniform by default.

Disadvantages:
  • Less flexible than Matplotlib for non-standard chart types; unusual
    layouts often require dropping back to the underlying Matplotlib API.
  • Implicit data aggregation can be surprising: seaborn may compute
    means or counts automatically when the developer expects raw values.
  • Fewer chart types available compared to Matplotlib (no 3D, no
    polar, limited inset support).
  • Depends entirely on Matplotlib – any seaborn environment includes
    Matplotlib as a transitive dependency.

Complexity:
  Low for standard statistical charts. Increases when customising
  beyond the built-in themes.

Flexibility / Configuration:
  Moderate – excellent for statistical and categorical plots; less
  suited for highly customised technical diagrams.

CONCLUSION

For IoT sensor monitoring applications, the two libraries complement
each other naturally. Matplotlib excels at rendering precise time-
series charts where multi-axis control, threshold annotations, and
exact visual placement matter. Seaborn is the better choice when the
goal is quickly communicating the statistical distribution of
categorical outcomes (e.g. how often was the environment "Comfortable"
versus "High"?). Together they provide a complete picture: Matplotlib
reveals the continuous dynamics, while Seaborn reveals the aggregate
classification behaviour.

If a third library were to be considered, Plotly would be the natural
next choice for interactive, browser-based dashboards that allow
zooming and tooltip inspection of individual readings.
